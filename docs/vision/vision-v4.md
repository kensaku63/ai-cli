# ai-cli ビジョン v4 統合: 「翻訳エンジン」から「能力解放」へ

**日付**: 2026-04-09
**著者**: make-vision.kensaku63
**背景**: kensaku63 との対話から、ai-cli の本質的価値を「翻訳品質」より大きなスケールで再定義

---

## 出発点: bash tool が変えた世界

bash tool は AI エージェントに**ツール統合コストゼロ**をもたらした。MCP サーバーを書く必要はない。プログラムをインストールした瞬間、AI のツールになる。

これは Web がコンテンツ公開コストをゼロにしたのと同じ構造的変化。

---

## 発見: 能力と知識の巨大なギャップ

### bash の先にある実行能力の全体像

| ソース | 規模 | 実行方法 |
|---|---|---|
| apt (Debian/Ubuntu) | 60,000+ | apt install → 実行 |
| npm (npmjs.com) | 2,000,000+ | npx で即実行（インストール不要） |
| PyPI (pip) | 500,000+ | pipx で即実行 |
| Homebrew | 6,000+ | brew install → 実行 |
| Go modules | 数万 | go install → 実行 |
| Cargo (Rust) | 150,000+ | cargo install → 実行 |
| Docker Hub | 数百万 | docker run で即実行 |
| GitHub Releases | 無数 | curl → 実行 |
| HTTP API (curl) | 世界中の全API | curl 一発 |

**合計: 数百万のプログラム + 世界中の API**

### さらに、合成が指数的な能力を生む

Unix パイプによる合成:
- 単体: 数百万の能力
- 2段階合成 (A | B): 数兆の組み合わせ
- 3段階以上: 事実上無限

```bash
curl API | jq '.data' | xargs -I{} convert {} output.png | imgcat
```

個々のプログラムの創造者が想定しなかった組み合わせが、新しい能力を生む。

### ギャップの実態

- AI が知っているツール: 数百（訓練データに頻出するもの）
- bash で実行可能なプログラム: 数百万
- 合成を含めた実行可能な操作: 事実上無限
- **AI の活用率: 0.01% 以下**

AI エージェントは持っている能力の 0.01% しか使えていない。最適なツールを知らないため、常に「次善の策」で動いている。

## 本質の再定義

### v3 の限界

v3 ビジョン「自然言語を最小コストで CLI 実行に変換する翻訳エンジン」は:
- **手段の説明**にとどまっている（翻訳は手段であり目的ではない）
- **スケール感が小さい**（数百の既知コマンドへの変換に聞こえる）
- **真の価値を表現できていない**（AI の潜在能力の解放）

### v4 の核心

**AI エージェントは bash tool によって無限の実行能力を手にした。しかし、何が使えるかを知らないため、その能力を活かしきれていない。ai-cli は、AI の知性に人類のソフトウェア資産全体を接続し、その潜在能力を解放する。**

```
AI の知性（考える力）
  × 人類のソフトウェア資産（数百万のプログラム）
  × Unix 的合成（パイプ、スクリプト）
  = 任意の問題を解く能力
```

ai-cli が提供するのは、この掛け算の「×」の部分。**接続**。

## v3 → v4 の変更点

| 要素 | v3（翻訳エンジン） | v4（能力解放） |
|---|---|---|
| 核心メタファー | 翻訳 | 接続・解放 |
| 価値の対象 | 既知コマンドへの変換 | 未知を含む全プログラムの活用 |
| スケール | CLI コマンド数百 | パッケージ数百万 + 合成 = 無限 |
| 差別化の源泉 | 翻訳品質 | ソフトウェア資産全体への接続 |
| 「安全性」の位置づけ | 方針の1つ | 能力解放に不可欠な信頼基盤 |
| 背景にある問い | どう変換するか？ | AI の潜在能力をどう解放するか？ |
| 射程 | CLI の便利ツール | AGI への構成要素 |
