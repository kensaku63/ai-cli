## 本質的価値

bash tool は AI エージェントに無限の実行能力を与えた。しかし AI は数百万のプログラムのうち、訓練時に学んだ数百のツールしか知らない。活用率は 0.01% 以下。

ai-cli は、この「能力と知識のギャップ」を埋める。AI の知性に人類のソフトウェア資産全体を接続し、潜在能力を解放する。

```
AI の知性（考える力）
  × 人類のソフトウェア資産（数百万のプログラム）
  × Unix 的合成（パイプ、スクリプト）
  = 任意の問題を解く能力
```

ai-cli が提供するのは、この掛け算の「×」の部分。**接続**。

## 仮説

- **H1: AI の潜在能力は実行能力ではなく知識で制約されている** — bash tool で実行能力は無限になった。ボトルネックは「何が使えるか」の知識
- **H2: 合成が指数的な能力を生む** — 数百万のプログラムを Unix パイプで合成すれば、事実上無限の操作が可能になる
- **H3: ターミナルは LLM 時代の最適インターフェース** — テキストイン・テキストアウトの一致。回帰ではなく必然
- **H4: 信頼が能力解放のゲートである** — 能力が大きいほど、安全性と透明性が普及の鍵になる

## 方針

1. **AI が知らないツールを知っている存在になる** — レジストリの質と規模で、AI の実効的な能力を拡張する
2. **合成パターンの知識を持つ** — 単体ツールだけでなく、ツール間の組み合わせ方を知っている
3. **安全性は能力解放の前提** — 能力が大きいほど信頼の設計が重要。信頼の型システム（Safe/Reversible/Destructive/Critical）
4. **Zero config, Single binary** — インストールした瞬間に価値を届ける
5. **Unixと共存する** — 既存のエコシステムを置き換えず、知識のレイヤーを重ねる
6. **オープンソースで育てる** — 世界中の開発者と共にソフトウェア資産の接続を拡張する
